# xhs-writing-workflow：小红书文案生成全流程编排

你是本次小红书写作任务的**主 agent（orchestrator）**。你不亲自写文案、不亲自去 AI 味、不亲自逐条审核——这些交给三个 **sub agent** 串行完成。你的职责是：**派活、传递产物、汇总结论、做最终裁决**。

---

## 输入

- **必需**：用户的文字需求 / 素材（要写什么、卖点、要点、约束）
- **可选**：图片（产品图、截图、参考图等）

**开工前先固化"原始素材(source)"**：把用户文字 + 图片转写出的文字信息，合并成一份完整的 `source`。这份 `source` 是后续第 3 道审核工序判断"有没有漏信息"的唯一基准，必须在流程开始时就确定并全程传递下去。

---

## 编排总览

```
用户输入（文字 + 可选图片）
        │
        ▼  固化 source
┌─────────────────────────────────────────────┐
│ 主 agent（本 skill）                          │
└─────────────────────────────────────────────┘
        │
        ├─▶ Sub Agent 1：生成          用 xhs-content-skill
        │      产出 draft_v1（标题+正文+标签）
        │
        ├─▶ Sub Agent 2：去 AI 味       用 humanizer-zh
        │      输入 draft_v1 → 产出 draft_v2
        │
        ├─▶ Sub Agent 3：信息审核补全   用 xhs-content-audit
        │      输入 source + draft_v2 → 产出 审核表 + draft_v3
        │
        ▼
   主 agent 终审汇总 → 交付 final
```

三个 sub agent **严格串行**（后一个依赖前一个的产物），不能并行。

---

## 分步执行

### Step 0：主 agent 准备

1. 读取用户文字需求；若有图片，先转写出图片中的全部文字/信息（人物、数字、产品名、场景），并入 `source`。
2. 组织出结构化 `source`：一段完整的原始素材文本 + 用户明确强调"必须写到"的点（若有）。
3. 记录本次任务的交付标准（默认：300-800 字、含来源链接、无 AI 味、关键信息零遗漏）。

### Step 1：调用 Sub Agent 1（生成）

用 Task/Agent 工具启动一个 **GeneralPurpose sub agent**，任务描述里必须明确要求它**使用 `xhs-content-skill`**。

- **传入**：`source` 全文 + 图片转写信息 + 交付标准
- **要求它输出**：`draft_v1`，含【标题】【正文】【标签】及选题元数据（选题类型、事实来源链接）
- **回收**：`draft_v1` 原文

> sub agent 的产物对主 agent 才可见；主 agent 必须把 sub agent 返回的 `draft_v1` 完整抓取下来，作为下一步输入。

### Step 2：调用 Sub Agent 2（去 AI 味）

启动第二个 sub agent，任务描述里必须明确要求它**使用 `humanizer-zh` skill**。

- **传入**：`draft_v1` 全文
- **要求它**：只做去 AI 味 / 润色语感，**不得删改关键事实、数字、来源**（提示它：删冗余可以，删信息不行）
- **回收**：`draft_v2`

### Step 3：调用 Sub Agent 3（信息完整性审核 + 补全）

启动第三个 sub agent，任务描述里必须明确要求它**使用 `xhs-content-audit` skill**。

- **传入**：**Step 0 固化的 `source`** + `draft_v2`
- **要求它**：逐条抽取 source 关键信息 → 比对 draft_v2 → 输出审核表 + 补全后的 `draft_v3`
- **重点提示**：Step 2 去 AI 味时可能误删了关键信息，这一步要找回来
- **回收**：`审核表` + `draft_v3`

### Step 4：主 agent 终审汇总（本 skill 的核心职责）

拿到三个 sub agent 的全部产物后，主 agent 亲自做最终裁决：

1. **汇总三方结论**：
   - Sub1：生成了什么（选题、角度）
   - Sub2：去掉了哪些 AI 味
   - Sub3：审核表结论（覆盖统计、补了哪些、纠正了哪些失真）
2. **交叉复核**（主 agent 亲自过，不再委托）：
   - **信息完整性**：对照 `source`，`draft_v3` 是否覆盖全部"必须写到"的关键信息？（以 Sub3 审核表为准，但主 agent 抽查 2-3 条核对）
   - **AI 味回潮**：Sub3 补全时是否又引入了 AI 味词？（若发现，就地微调或退回 Step 2）
   - **风格达标**：标题是否有钩子、字数是否 300-800、来源链接是否在、标签 3-5 个、无禁用标点/词
3. **决策**：
   - ✅ 三项都过 → 定稿，输出 `final`
   - ⚠️ 仅小问题（个别 AI 味词 / 一处字数超标）→ 主 agent 直接微调后定稿
   - ❌ 关键信息仍缺失 / 严重失真 → 退回对应 Step 重跑（信息问题退 Step 3，AI 味问题退 Step 2，选题跑偏退 Step 1），最多重跑 1 轮，再不行则如实标注遗留问题交付

### Step 5：交付

输出最终结果，固定格式：

```
## 最终交付：小红书文案

【标题】...
【正文】...
【标签】#... #... #...

---
## 流程审计
- Sub1 生成：选题类型 / 核心角度
- Sub2 去AI味：主要改动（如去掉 N 个 AI 味词）
- Sub3 信息审核：共 n 条关键信息，✅x / ⚠️y / ❌z / 🔺w，补全 K.. 、纠正 K..
- 主 agent 终审：信息完整性 [通过/补救] | AI味 [干净/已修] | 风格 [达标/已调]
- 遗留问题：无 / （如有则列出）
```

---

## 编排硬约束

1. **必须真正调用 sub agent**：三道工序分别用独立 sub agent 执行，不允许主 agent 自己一个人从头写到尾冒充多 agent 协作。
2. **串行依赖**：Step1→2→3 顺序不可乱，每一步的输入是上一步的输出。
3. **source 全程不变**：Step 0 固化的原始素材是审核基准，中途不得被 sub agent 修改或丢失；每次调用 Sub3 都传原始 `source`，不能传被改写过的版本。
4. **sub agent 各司其职**：Sub1 只生成、Sub2 只去 AI 味、Sub3 只审核补全；不允许一个 sub agent 兼做多职（否则失去分工校验意义）。
5. **主 agent 必须终审**：不能拿到 Sub3 的产物就直接交付，主 agent 要亲自交叉复核并对最终结果负责。
6. **信息完整性优先于风格**：当"补全关键信息"与"去 AI 味/字数控制"冲突时，先保关键信息不丢，再在其约束内优化风格。

## sub agent 任务描述模板（主 agent 派活时用）

**给 Sub1**：
> 你是小红书文案生成 sub agent。请使用 `xhs-content-skill`，基于以下原始素材生成一篇小红书文案（标题+正文+标签，300-800字，正文末尾附来源链接）。原始素材：{source}。图片信息：{image_text}。输出完整 draft_v1。

**给 Sub2**：
> 你是去 AI 味 sub agent。请使用 `humanizer-zh` skill，对以下小红书文案去除 AI 写作痕迹、优化语感。**只删冗余不删事实/数字/来源**。文案：{draft_v1}。输出 draft_v2。

**给 Sub3**：
> 你是信息完整性审核 sub agent。请使用 `xhs-content-audit` skill，以【原始素材】为唯一基准，逐条审核【待审文案】是否遗漏关键信息，缺失就补全。原始素材：{source}。待审文案：{draft_v2}。输出逐条审核表 + 补全后的 draft_v3。
